# JOB-09 v0.2.7 — RLS Progression + Personal Claims Change Summary

**Baseline:** v0.2.6 Police Impound + Emergency Scenes  
**Candidate:** v0.2.7 RLS Progression + Personal Claims  
**Status:** BUILT — RUNTIME UNTESTED

## Main runtime changes

### Progression calculation

Added a progression calculation layer that derives:

- workload units by Tow event type;
- distance contribution;
- target-count contribution;
- Beam XP;
- Labourer progression;
- Repo XP when `careerSkills-repo` exists;
- Civil Service XP for police/enforcement event types when available.

### Money calculation

Retained existing distance-sensitive pay and added explicit workload/target scaling. When RLS exposes the relevant systems, the final pay calculation also consumes:

- Repo skill-level bonus;
- `repo` economy multiplier;
- difficulty reward scaling.

### Pending-payment persistence

Expanded queued payment records to preserve progression and event metadata. The processing path remains backwards-compatible with older queued records that contain money only.

### Inventory tow-vehicle progression

The inventory ID of the player's tow vehicle is captured when a call is accepted. On successful reward processing, the RLS `addRepossession` statistic is incremented once per delivered target when the API exists.

### Tow History and active-call UI

Added fields for:

- workload;
- Repo bonus;
- Beam XP;
- Labourer progression;
- Repo XP;
- Civil Service XP.

### Tow Yard personal claim

Added `claimYardVehicleToCareerInventory(recordId)` and a corresponding yard UI action.

The implementation:

1. validates hold expiry;
2. validates Career inventory and garage capacity;
3. retrieves/spawns the yard vehicle;
4. adds it to inventory as owned;
5. marks RedFox claim metadata;
6. moves it to a private garage;
7. requests part-condition capture;
8. removes the live object after storage;
9. removes the yard record only after success;
10. rolls back and preserves the yard record after failure.

No sale payment is issued.
