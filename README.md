# RedFox Master Control

This repo is the shared communication bridge for David's RedFox BeamNG mod rebuild.

---

## One File First

Every RedFox chat should start with this one file:

```text
RedFox_Worker_Chat_Quick_Start.md
```

That file explains:

```text
1. How the chat finds its assigned role.
2. Which extra files to read only if needed.
3. How to leave a written footprint.
4. What to do if the chat does not know its assignment.
```

David should not have to explain the whole system every time.

---

## Main Communication Rule

Reading the repo is not enough.

Every worker chat must leave a written footprint by doing one of these:

```text
1. Update RedFox_Chat_Message_Board.md directly, or
2. Give David an exact message-board block so the Coordinator Chat can post it.
```

If there is no message-board/log entry, communication failed.

---

## Important Files

| File | Purpose |
|---|---|
| RedFox_Worker_Chat_Quick_Start.md | One-file start instructions for every chat. |
| RedFox_Chat_Message_Board.md | Shared message board. Every chat must leave something here or return a block. |
| RedFox_Master_Build_Control_Document.md | Full RedFox project rules, bridge contracts, module registry, and roadmap. |
| RedFox_File_Verification_Checklist.csv | Before/after file editing checklist. |
| RedFox_Test_Results_Table.csv | BeamNG testing result tracker. |
| RedFox_Module_Status_Table.csv | Module status tracker. |
| RedFox_Communication_SignIn_Log.md | Detailed sign-in, work-update, and finish log. |

---

## Repo Rule

No worker chat should edit real BeamNG mods until its assigned role is clear and it has left or returned a message-board entry.

No fake verification. No silent renames. No silent reads. No giant messy upload piles.
